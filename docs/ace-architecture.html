Layer 3 — ACE Vault (Nikhil's System — R&D Lab)
Vault Structure
Deterministic routing via context-router.md. Each data type has one canonical home.
Actualize/
├── 00-System/ ← state, active, routing, schemas, permissions
│   ├── core/ ← identity (DCA, triad, voice, edges) — human-edit only
│   ├── state.md ← mode, energy, outcomes, failures
│   ├── active.md ← max 3 current outcomes
│   └── context-router.md ← placement rules
├── 01-Journal/ ← daily notes, patterns, weekly reviews
├── 02-Rituals/ ← coaching session log
├── 03-Calendar/ ← commitments (rules only, not events)
├── 04-Network/ ← people, follow-ups, network map
├── 05-Research/ ← topics, finance, reference
├── Domains/ ← actualize, adept-arcanum, asraya, ...
│   └── actualize/clients/ ← per-client build materials
├── .claude/skills/ ← 30+ command specs
└── 11-Artifacts/ ← shipped creative work
Data Architecture
Markdown for identity & narrative. SQLite for metrics & pipeline. Git for history.
Identity Layer
00-System/core/ — DCA, coherence triad, nervous system map, coach constitution, voice profile, edges & fears. Human-authored. Never AI-generated.
markdown · wikilinks · human-edit only
Operational Layer
state.md + active.md + decision-log + execution-log + follow-ups. AI reads and writes within permission bounds.
markdown · schema-validated
Metrics Layer
ace.db (SQLite) — pipeline deals, revenue events, campaign stats, social metrics. Single source of truth for numbers.
better-sqlite3 · ace-analytics MCP
Memory Layer
File-based auto-memory (MEMORY.md index) + Hindsight semantic recall. Behavioral rules, people facts, project state.
markdown + Hindsight MCP
